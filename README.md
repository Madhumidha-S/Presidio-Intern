# Presidio Internship Repository

Welcome to the Presidio Internship repository. This repository is structured across multiple branches to maintain a clean separation of learning notes, projects, and weekly challenges.

## Repository Structure

Instead of maintaining all files on the `main` branch, this repository utilizes a branch-based organization strategy. Each topic, challenge, or week has its own dedicated branch containing specific notes, project files, and its own context-specific README.

### Weekly Logs
These branches contain progress, notes, and tasks completed during specific weeks:
- `week1`
- `week2`
- `week4`
- `week5`

### Challenges
These branches contain solutions and documentation for internship challenges:
- `challenge1`
- `challenge2`
- `challenge3`
- `challenge4`
- `challenge5`

### Technical Modules & Features
These branches contain dedicated markdown notes and implementation details on specific technical topics:
- `feature/api` - API concepts
- `feature/async` - Asynchronous programming
- `feature/auth` - Authentication mechanisms
- `feature/cicd` - CI/CD pipelines
- `feature/dbms` - Database Management Systems
- `feature/dsa/hashmap` - Hashmap Data Structure
- `feature/genai` - Generative AI
- `feature/git` - Git & Version Control
- `feature/iac` - Infrastructure as Code
- `feature/log` - Logging strategies
- `feature/middleware` - Middleware implementation
- `feature/recursion` - Recursion concepts
- `feature/comment-parser` - Comment parser project
- `feature/hashtag-engine` - Hashtag engine project

## How to Navigate

To access the notes or code for a specific module, switch to the corresponding branch.

**Using Git (Command Line):**
```bash
git fetch --all
git checkout <branch-name>
# Example: git checkout feature/auth
```

**Using the Web Interface (e.g., GitHub/GitLab):**
Select the branch dropdown menu at the top left of the repository file list and choose the branch you wish to explore.

## Branch-Specific READMEs

This `main` branch README serves as the central directory for the entire repository. 

**What to do with other branch READMEs:**
The `README.md` files located in the individual branches should remain specific to their respective topics. For example, the README in `feature/auth` should only explain the authentication notes and code present in that branch. You do not need to update them unless you are modifying the contents of that specific branch.
